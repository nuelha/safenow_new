#### 경고 배너 노출 조건

```
standard_documents.is_required = true 인 항목 중
해당 company_id의 documents에 ACTIVE 레코드 없는 건수 > 0
→ 배너 노출
→ [미등록 문서 확인] 클릭 시 status 필터 = MISSING 자동 적용
```

### DOC01-M2 새 버전 등록 모달

| 항목 | 내용 |
|------|------|
| 화면 ID | DOC01-M2 |
| 접근 권한 | SHM+ |

#### 구성

```
현재 버전 정보 (읽기전용)
문서명: {document_name}  |  현재 버전: {version}

─────────────────────────────────────
[파일형] 새 파일 업로드 영역
[데이터입력형] 기존 데이터 불러와 수정

버전 번호: [v2.2] (현재+0.1 자동 제안, 수정 가능)
변경 사유: [필수 입력]

[취소]  [새 버전 등록]
```

#### 저장 동작

```
새 버전 등록 클릭:
1. document_versions INSERT
2. documents.version 업데이트
3. documents.status = 'ACTIVE' (NEEDS_UPDATE → ACTIVE 복귀)
4. documents.updated_at 갱신
5. 이행관리 연결된 경우: compliance_tasks 완료 처리
```

#### 중복 문서명 처리

```
동일 카테고리 + 동일 문서명으로 + 새 문서 등록 시도:
→ "이미 등록된 문서가 있습니다. 새 버전을 등록하시겠습니까?" 팝업
→ 예: DOC01-M2로 이동 / 아니오: 계속 등록
```

2. 확인 클릭:
   a. documents INSERT
      - document_name: "{표준문서명} (가져옴)"
      - source_standard_doc_id: 표준문서 ID
      - version: "v1.0" / status: ACTIVE
      - management_type: 원본과 동일
      - form_schema: 원본 복사 (데이터입력형인 경우)
   b. DOC01-D 상세로 자동 이동

표준문서 원본 업데이트 시:
→ source_standard_doc_id 참조 모든 레코드의 status = NEEDS_UPDATE
→ 개정필요 배너에 반영
```

### DOC03-F 문서 등록 (멀티스텝 모달)

#### STEP 1. 관리방식 선택

```
┌──────────────────────┐  ┌──────────────────────┐
│  📄 파일형            │  │  📝 데이터입력형       │
│                      │  │                      │
│  PDF, Word 등 파일   │  │  양식을 구성하고       │
│  업로드해서 관리       │  │  항목별 직접 입력      │
│                      │  │                      │
│  절차서, 방침서,       │  │  평가표, 회의록,       │
│  계획서, 신고서        │  │  기록부, 확인서        │
└──────────────────────┘  └──────────────────────┘
```

#### STEP 2-A. 파일형 — 파일 업로드

| 항목 | 규칙 |
|------|------|
| 지원 형식 | PDF / Word(.doc, .docx) / Excel(.xls, .xlsx) / PPT(.ppt, .pptx) / 이미지(JPG, PNG) |
| 최대 용량 | 50MB |
| ⚠ 제거 | 기존 "표준문서에서 선택" UI 없음. 표준문서는 DOC02-L에서만 접근 |

#### STEP 2-B. 데이터입력형 — 문서 유형 선택 및 양식 구성

**문서 유형 5종**

| 유형 코드 | 표시명 | 특징 | 연동 |
|---------|--------|------|------|
| `evaluation` | 평가표 | 점수 자동합산 + 적격/부적격 자동판정 | 도급관리 평가서, 인력평가 |
| `minutes` | 회의록 | 안건별 협의 + 참석자 서명 | — |
| `log` | 기록부 | 반복 행 추가 테이블 | 안전보건교육 기록 |
| `confirmation` | 확인서 | 서명 필수. 서명완료 시 PDF 잠금 | 도급 서약서 |
| `checklist` | 체크리스트 | 적합/부적합/해당없음 + 개선조치 연동 | 안전점검 점검지 |

**양식 구성 UI**

```
[+ 섹션 추가]

섹션 1. {섹션명}  [삭제]
  항목 1. {항목명}  [유형▼]  [필수☑]  [삭제]
  항목 2. {항목명}  [유형▼]  [필수☑]  [삭제]
  [+ 항목 추가]

[미리보기]  [다음 →]
```

**항목 유형 코드 정의**

| 유형 코드 | 표시명 | 설명 |
|---------|--------|------|
| `text` | 텍스트 | 단행 입력 |
| `textarea` | 텍스트(장문) | 여러 줄 |
| `number` | 숫자 | |
| `date` | 날짜 | |
| `datetime` | 날짜+시간 | |
| `select` | 선택 | 단일 선택 |
| `checkbox` | 체크박스 | 예/아니오 |
| `score` | 점수 | 숫자 + 만점 + 척도 |
| `inspection` | 점검결과 | 적합/부적합/해당없음 |
| `signature` | 전자서명 | |
| `user` | 사용자 선택 | 조직도 검색 |
| `file` | 파일첨부 | |

**평가표 전용 설정 (evaluation_config)**

| 설정 | 설명 |
|------|------|
| 만점 | 전체 배점 합계 |
| 적격 기준점수 | 이 점수 이상이면 적격 |
| 평가 척도 | 5단계 / 3단계 / 2단계 / 직접 점수 입력 |

#### STEP 3. 문서 정보 (공통)

| 필드 | 타입 | 필수 | 기본값 |
|------|------|------|------|
| 문서명 | TEXT | ✅ | — |

| 법적 근거 | TEXT | ❌ | — |
| 설명 | TEXTAREA | ❌ | — |
| 태그 | MULTI_TEXT | ❌ | — |

#### STEP 4. 권한 설정 (공통)

```

1. documents INSERT

## 2. 공정관리 (PRC)

### PRC01-L 공정 목록

| 항목 | 내용 |
|------|------|
| 화면 ID | PRC01-L |
| URL | `/process` |
| 접근 권한 | SHM+ |
| 기본 진입 화면 | ✅ |

#### 레이아웃

```
┌────────────────────────────────────────────────────┐
│ 공정관리                                           │
│                                                    │
│ 필터: [사용여부▼]                       [+ 등록]    │
│                                                    │
│ 공정명 | 설명 | 작업수 | 사용여부(토글) | 수정 | 삭제│
└────────────────────────────────────────────────────┘
```

#### processes 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `site_id` | UUID FK | |
| `name` | VARCHAR(100) | 공정명 |
| `description` | TEXT | |
| `is_active` | BOOLEAN | 사용여부 |
| `created_at` | TIMESTAMP | |

#### process_tasks 테이블 (공정 내 작업)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `process_id` | UUID FK | |
| `name` | VARCHAR(100) | 작업명 |
| `description` | TEXT | |
| `sort_order` | INTEGER | 순서 |

#### 사용여부 토글 동작

```
토글 ON/OFF:
processes.is_active 업데이트

is_active = false 인 공정:
- TBM 그룹 공정 연결 선택 목록에서 제외
- 위험성평가 공정 연결 선택 목록에서 제외
```

### PRC01-M 공정 등록 모달

| 항목 | 내용 |
|------|------|
| 화면 ID | PRC01-M |
| 접근 권한 | SHM+ |
| 진입 경로 | PRC01-L > + 등록 버튼 |

#### 입력 항목

| 필드 | 타입 | 필수 |
|------|------|------|
| 공정명 | TEXT | ✅ |
| 설명 | TEXTAREA | ❌ |

---

### PRC01-M2 공정 수정 모달

| 항목 | 내용 |
|------|------|
| 화면 ID | PRC01-M2 |
| 접근 권한 | SHM+ |
| 진입 경로 | PRC01-L 수정 버튼 |

#### 구성

```
기본 정보
  공정명 / 설명

작업 목록
  ─────────────────────────────────────
  1. {작업명}  {설명}  [↑][↓][삭제]
  2. {작업명}  {설명}  [↑][↓][삭제]
  ─────────────────────────────────────
  [+ 작업 추가]
```

#### 작업 항목 입력

| 필드 | 타입 | 필수 |
|------|------|------|
| 작업명 | TEXT | ✅ |
| 설명 | TEXT | ❌ |

#### 저장 동작

```
저장 클릭:
1. processes 업데이트
2. process_tasks 전체 교체 (DELETE + INSERT)
   sort_order: 목록 순서 기준 자동 설정
```
