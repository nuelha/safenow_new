## 0. 진입 플로우 (LOGIN-S)

### LOGIN-S 사업장 선택 화면

| 항목 | 내용 |
|------|------|
| 화면 ID | LOGIN-S |
| URL | `/site-select` |
| 접근 권한 | 로그인 완료 사용자 (전체) |
| 진입 조건 | 로그인 완료 시 항상 (대시보드 직행 차단) |
| 뒤로가기 | 입장 후엔 로그인 화면으로만 복귀 (브라우저 히스토리 replace 처리) |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 🛡️ 안전NOW                            박안전 님 ∨ │
├────────────────────────────────────────────────────┤
│ 사업장                                              │
│ 접속할 사업장을 선택해 주세요                        │
│                                                    │
│ [🔍 사업장 검색]                                    │
│                                                    │
│ No │ 사업장명         │ 구분       │ 권한      │ 알림 │ 상태│
│  1 │ 다온산업 본사    │ 구독사업자  │ 경영책임자  │ 토글 │[입장]│
│  2 │ 다온산업 평택공장 │ 구독사업자  │ 안전보건관리자│ 토글 │[입장]│
│  3 │ 동방기계㈜ 현장   │ 도급업체    │ 도급담당자   │ 토글 │[입장]│
│  4 │ (주)대한안전컨설팅 │ 컨설팅업체  │ 컨설팅담당자 │ 토글 │[입장]│
└────────────────────────────────────────────────────┘
```

#### 컬럼 정의

| 컬럼 | 데이터 소스 | 비고 |
|------|-----------|------|
| No | 순번 | 검색 필터 후 재계산 |
| 사업장명 | `companies.company_name` + `sites.name` | 회사명/사업장 2줄 |
| 구분 | `companies.company_type` | 배지 |
| 권한 | `memberships.role` | 역할 표시명 |
| 알림 수신 | `membership_notification_settings.kakao_enabled` | 카카오톡 토글 |
| 상태 | [입장] 버튼 | 클릭 시 라우팅 |

#### 구분 배지

| company_type | 표시명 | 색상 |
|-------------|--------|------|
| `principal` | 구독사업자 | 파랑 |
| `subcontractor` | 도급업체 | 주황 |
| `consulting` | 컨설팅업체 | 보라 |

#### 입장 라우팅 분기

| company_type | role | 진입 화면 |
|-------------|------|---------|
| principal | CEO | DSH01-V |
| principal | GM/SM/SHM | DSH02-V |
| principal | WKR | DSH03-V |
| subcontractor | subcontractor_manager | **DSH04-V** |
| consulting | consulting_manager | **DSH04-V** |

#### 입장 시 세션 저장

```javascript
// 사이드바 워크스페이스 선택기와 호환되는 localStorage 키
safenow_role            // ceo | manager | worker | contractor | consulting
safenow_workplace       // 사업장명
safenow_user            // 사용자명
safenow_avatar          // 아바타 1글자
safenow_company_type    // principal | subcontractor | consulting
safenow_company_name    // 회사명
```

- 입장 시 `window.location.replace(target)`로 이동 (뒤로가기 우회 방지)

#### 컨텍스트 전환 동선

```
GNB 하단 사업장 선택기 클릭
→ /site-select (이 화면)으로 이동
→ 다른 사업장 [입장] 클릭
→ 새 컨텍스트로 진입 (로그아웃 없이 전환)
```

## 1. 대시보드 (DSH)

### DSH01-V 경영책임자 대시보드

| 항목 | 내용 |
|------|------|
| 화면 ID | DSH01-V |
| URL | `/` (기본 진입) |
| 접근 권한 | CEO |
| 진입 경로 | 로그인 후 자동 진입 / GNB > 대시보드 |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 이행률 요약 카드 (카드 전체 클릭 → CMP01-V)          │
│ 완료 N건 | 진행중 N건 | 계획 N건 | 전체 이행률 %    │
│                                                    │
│ [목표/KPI 달성률] [결재대기 목록]                    │
│                                                    │
│ [아차사고·작업중지 추이] [개선조치 현황]             │
│                                                    │
│ [의견청취 현황]   [교육 현황]                       │
└────────────────────────────────────────────────────┘
```

> ※ 내 할일 위젯은 CEO 대시보드에서는 노출하지 않음 (관리자/근로자 대시보드에만 노출).

#### 위젯별 데이터 정의

**이행률 요약 카드**

| 표시 항목 | 데이터 소스 | 산출 방식 |
|----------|-----------|---------|
| 완료 건수 | `compliance_tasks.status = 'DONE'` | COUNT |
| 진행중 건수 | `compliance_tasks.status = 'IN_PROGRESS'` | COUNT |
| 계획 건수 | `compliance_tasks.status = 'SCHEDULED'` | COUNT |
| 전체 이행률 | 완료 / 전체 × 100 | % |

- 조회 범위: 로그인한 사용자의 `company_id` 기준 전체 사업장
- 기간: 현재 연도 기준
- **카드 전체 클릭 → CMP01-V (이행관리)** 로 이동

**목표/KPI 달성률**

| 표시 항목 | 데이터 소스 | 비고 |
|----------|-----------|------|
| KPI명 | `safety_kpis.name` | |
| 달성률 | `safety_kpis.current / safety_kpis.target × 100` | % |
| 목표값 / 현재값 | `safety_kpis.target` / `current` | |

- [바로가기] → POL03-S (안전경영방침 > 목표·KPI 탭)

**결재대기 목록 (CEO 전용)**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 상신일 | `approvals.submitted_at` |
| 제목 | `approvals.title` |
| 상신자 | `users.name` (상신자) |
| 종류 배지 | `approvals.type` (예산·도급선정·인력평가·경영방침점검 등) |

- 노출 범위: 로그인한 CEO가 최종 결재자인 항목 중 상태 = `PENDING`
- 위치: 목표/KPI 달성률 카드 우측(2단 그리드)
- 정렬: 상신일 오름차순(오래된 건 우선)
- 행 클릭 → TSK03-L (결재대기 상세) 또는 결재 처리 모달
- [바로가기] → TSK03-L

**아차사고·작업중지 발생 추이**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 아차사고 | `opinion_requests.category = 'near_miss'` 월별 COUNT |
| 작업중지 | `opinion_requests.category = 'work_stop'` 월별 COUNT |

- 기간 필터: **6개월 / 12개월 토글** (선택 시 월별 데이터 재집계)
- 차트 유형: 월별 막대 차트(아차사고/작업중지 그룹 막대)
- **인터랙션**:
  - 막대(월) 클릭 → OPN01-L 진입 시 `?category=near_miss|work_stop&month=YYYY-MM` 파라미터 전달
  - 카드 자체에는 [바로가기] 링크 **없음** (그래프 클릭이 진입 동선)
- 차트만으로 진입 동선이 성립하므로 우상단 [바로가기] 버튼은 노출하지 않음

**개선조치 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 대기중 | `improvements.status = 'PENDING'` COUNT |
| 조치중 | `improvements.status = 'IN_PROGRESS'` COUNT |
| 조치완료 | `improvements.status = 'DONE'` COUNT |

- 목록: 최근 3건 카드 표시 (완료예정일 / 조치내용 / 담당자)
- [바로가기] → IMP01-L
- **행 클릭** → IMP01-L 진입 시 상태 배지에 따라 `?status=pending|action|done` 자동 필터 적용

**의견청취 현황**

| 표시 라벨 | 의미 | 데이터 소스 (`opinion_requests.status`) | URL `status=` 값 |
|----------|------|---------------------------------------|------------------|
| 접수대기 | 처리 대기 중 | `RECEIVED` | `pending` |
| 점검중 | 안전점검 진행 중 | `INSPECTING` | `inspection` |
| 개선조치중 | 개선조치 진행 중 | `IMPROVING` | `improvement` |
| 반려 | 반려 처리됨 | `REJECTED` | `reject` |
| 완료 | 처리 완료 | `DONE` | `done` |

- 표시 라벨은 OPN01-L 화면(상태 필터 옵션) 라벨과 정확히 일치해야 함
- 분류별 배지 필터 (청취조사 / 안전제안 / 위험신고)
- 목록: 최근 3건 카드 표시
- [바로가기] → OPN01-L
- **분류 탭 클릭** → OPN01-L `?category=survey|suggestion|hazard_report` 진입
- **행 클릭** → OPN01-L 진입 시 상태 배지에 따라 `?status=` 자동 필터 적용

**교육 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 교육명 | `safety_education_schedules.name` |
| 이수율 | `완료 인원 / 대상 인원 × 100` % |

- [바로가기] → EDU01-V
- **행 클릭** → EDU01-V로 이동

#### 연계 화면

| 위젯 | 연계 화면 | 진입 방식 |
|------|---------|----------|
| 이행률 요약 카드 | CMP01-V | 카드 전체 클릭 |
| 목표/KPI | POL03-S | 바로가기 |
| 결재대기 목록 | TSK03-L | 행 클릭 / 바로가기 |
| 아차사고·작업중지 추이 | OPN01-L | 막대 클릭(월·분류 파라미터 전달) |
| 개선조치 | IMP01-L | 바로가기 |
| 의견청취 | OPN01-L | 바로가기 |
| 교육현황 | EDU01-V | 바로가기 |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 내 교육 현황 (분기별 달성률)                        │
│                                                    │
│ [내 할일]                                          │
│                                                    │
│ 바로가기 버튼                                       │
│ [아차사고 신고] [작업중지권 행사] [안전제안]          │

```

#### 위젯별 데이터 정의

**내 교육 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 달성률 | `(TBM시간 + 교육시간) / 법정이수시간 × 100` % |
| 법정 이수시간 | `legal_education_hours.required` |
| 정기교육 이수시간 | `education_records.hours` WHERE `user_id = 본인` |
| TBM 인정시간 | `tbm_records.duration` WHERE `user_id = 본인` |

- 기간 탭: 26년 1분기 / 26년 2분기 / 25년 4분기

**바로가기 버튼 동작**

| 버튼 | 동작 |
|------|------|
| 아차사고 신고 | OPN01-F (category='near_miss' 자동 선택) |
| 작업중지권 행사 | OPN01-F (category='work_stop' 자동 선택) |
| 안전제안 | OPN01-F (category='suggestion' 자동 선택) |

#### 대시보드 뷰 전환 로직

```
로그인한 사용자의 role 확인
- role = 'CEO' → DSH01-V 노출
- role in ('GM', 'SM', 'SHM') → DSH02-V 노출
- role in ('WKR', 'SUB') → DSH03-V 노출

뷰 전환 진입점
→ 좌측 사이드바 하단 사업장/권한 선택기에서 다른 역할 옵션 선택 시 자동 전환
※ 헤더 상단의 역할 탭은 제거됨 (사업장 선택기로 일원화)
```

### DSH04-V 도급/컨설팅 간소 대시보드

| 항목 | 내용 |
|------|------|
| 화면 ID | DSH04-V |
| URL | `/dashboard/sub` (구현: `pages/dashboard-sub.html`) |
| 접근 권한 | SUB (도급담당자), CON (컨설팅담당자) |
| 진입 경로 | LOGIN-S 입장 시 `company_type ∈ {subcontractor, consulting}` 인 경우 자동 진입 |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ ⚠ 현재 작업 중인 원청 사업장: 다온산업 본사 현장      │ ← 컨텍스트 배너
│                                                    │
│ 안녕하세요, 강동방 님                                │
│ 동방기계㈜ · 도급업체 · 도급담당자                    │
│                                                    │
│ [내 할일 미니뷰]         [TBM 현황]                 │
│  미완료 N건               이번달 작성 N건             │
│  기한임박 N건             예정 N건                   │
│  최근 업무 2건            누적시간 / 분기 누적시간    │
│  [내 업무 전체보기→]      [TBM 작성→]               │
│                                                    │
│ 바로가기                                             │
│ [아차사고 신고] [안전제안] [위험신고]                 │
└────────────────────────────────────────────────────┘
```

#### 위젯별 데이터 정의

**컨텍스트 배너**

- 표시: `현재 작업 중인 원청 사업장` + `sites.name` (계약된 원청 사업장)
- 다중 계약인 경우 [사업장 전환 ▼] 드롭다운 노출
- 데이터 소스: 활성 계약 + `subcontractor_evaluations.contract_id`로 조회

**내 할일 미니뷰**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 미완료 | `tasks` WHERE `assignee_id = 본인` AND `status != 'DONE'` COUNT |
| 기한임박 | `tasks` WHERE `due_date - NOW() <= 3일` AND `status != 'DONE'` COUNT |
| 최근 업무 목록 | 최근 2건 (제목 + D-N 표시) |

- [내 업무 전체보기 →] → TSK01-L (`/my-tasks`)

**TBM 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 이번달 작성 | `tbm_records` WHERE `company_id = 본인사` AND `work_date IN current month` AND `status = 'DONE'` |
| 예정 | `tbm_records` WHERE `work_date > NOW()` AND `status = 'DRAFT'` |
| 이번달 누적시간 | `tbm_records.duration` SUM |
| 분기 누적시간 / 법정기준 | 분기 합계 + 법정 12h 비교 |

- [TBM 작성 →] → TBM01-F (`/tbm/new`)

**바로가기 (의견청취 작성 폼 분류별 진입)**

| 버튼 | 동작 |
|------|------|
| 아차사고 신고 | OPN01-F (`?category=near_miss`) |
| 안전제안 | OPN01-F (`?category=suggestion`) |
| 위험신고 | OPN01-F (`?category=hazard_report`) |

#### GNB 메뉴 (SUB/CON 컨텍스트)

| 컨텍스트 | 표시 메뉴 |
|---------|---------|
| subcontractor | 대시보드 / 내 업무 / TBM / 의견청취 / 고객지원 / 내 정보관리 |
| consulting | 대시보드 / 내 업무 / 의견청취 / 고객지원 / 내 정보관리 (위임 시 risk·compliance·documents·improvement 추가) |

- 사이드바 좌측 GNB는 `js/app.js > applyGnbByContext()`가 `safenow_company_type` localStorage 값으로 동적 필터링
- 사이드바 "대시보드" 링크 href는 컨텍스트별로 자동 재작성: principal → `index.html`, subcontractor/consulting → `dashboard-sub.html`
- DSH01~03 (구독사업자 전용 대시보드)에 SUB/CON이 직접 진입하면 자동으로 DSH04-V로 리다이렉트 (`index.html` 헤드 인라인 스크립트가 `safenow_company_type` 확인 후 `window.location.replace`)

#### 사업장 전환

- 사이드바 하단 사업장 선택기 클릭 → LOGIN-S 화면으로 이동 (재선택)
- 로그아웃 없이 컨텍스트 전환 가능

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 내 업무                                            │
│ 요약 카드: 해야할일 N | 기한초과 N | 완료 N | 진행률 % │
│                                                    │
│ [전체] [문서업무] [결재대기] [전자서명] [완료]        │ ← 탭
│                                                    │
│ 필터: [유형▼] [상태▼]                               │
│                                                    │
│ 기한초과 업무 N건 묶음 (조건부)                     │
│ ─────────────────────────────────────────────     │
│ 업무카드 목록                                       │
│ [유형배지] [상태배지] 업무명                         │
│ 사업장 · 담당자 · 기한                               │
│ [처리하기 / 상세보기 / 결재하기 / 서명하기]           │
│                                                    │
│ [페이지네이션]                                      │
└────────────────────────────────────────────────────┘
```

#### 요약 카드 데이터 정의

| 카드 | 데이터 소스 | 조건 |
|------|-----------|------|
| 해야할일 | `tasks` | `assignee_id = 본인` AND `status != 'DONE'` COUNT |
| 기한초과 | `tasks` | `assignee_id = 본인` AND `due_date < NOW()` AND `status != 'DONE'` COUNT |
| 완료 | `tasks` | `assignee_id = 본인` AND `status = 'DONE'` COUNT (최근 30일) |
| 진행률 | 완료 / 전체 × 100 | |

#### tasks 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `task_type` | ENUM | `evaluation` `tbm` `inspection` `education` `improvement` `compliance` `approval` `signature` `subcontractor_eval` |
| `title` | VARCHAR(200) | 업무명 |
| `assignee_id` | UUID FK | 담당자 |
| `reference_id` | UUID | 연결된 원본 레코드 ID |
| `reference_type` | VARCHAR(50) | 원본 테이블명 |
| `site_id` | UUID FK | 사업장 |
| `due_date` | DATE | 기한 |
| `status` | ENUM | `PENDING` `IN_PROGRESS` `DONE` `OVERDUE` |
| `created_at` | TIMESTAMP | |

#### 업무카드 구성 요소

| 요소 | 내용 |
|------|------|
| 유형 배지 | task_type 기반 (위험성평가 / TBM / 안전점검 / 개선조치 등) |
| 상태 배지 | 대기 / 진행중 / 기한임박 / 기한초과 / 완료 |
| D-N 표시 | 기한까지 남은 일수 (D-3, D-1, 오늘, 기한초과 N일) |

#### 업무 유형별 버튼 및 연계 화면

| 업무 유형 | 버튼 | 연계 화면 |

| 전자서명 | 서명하기 | 해당 원본 상세 |

| 필터 | 옵션 |
|------|------|
| 업무 유형 | 전체 / 위험성평가 / TBM / 안전점검 / 안전보건교육 / 개선조치 / 이행관리 |

#### 외부 진입 동선

| 진입처 | URL | 동작 |
|-------|-----|------|
| DSH01-V 결재대기 카드 [바로가기] | `/my-tasks?tab=approval` | 결재대기 탭 자동 활성화 |

- `?tab=approval` 진입 시 페이지 로드 후 자동으로 결재대기 탭이 선택된 상태로 표시
- `?id=` 가 함께 전달되면 해당 결재 카드로 스크롤 또는 모달 자동 오픈

|------|------|
| 업무명 | `approvals.title` |
| 상신일 | `approvals.submitted_at` |
| 상신자 | `users.name` (submitter_id join) |
| 버튼 | [반려] [승인] |

#### approvals 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `approval_type` | VARCHAR(50) | 결재 유형 |
| `title` | VARCHAR(200) | |
| `reference_id` | UUID | 원본 레코드 ID |
| `submitter_id` | UUID FK | 상신자 |
| `current_approver_id` | UUID FK | 현재 결재자 |
| `status` | ENUM | `PENDING` `APPROVED` `REJECTED` |

#### 결재 처리 동작

```
승인 클릭:
1. approvals.status = 'APPROVED'
2. 결재라인 다음 결재자에게 업무 배정
3. 마지막 결재자 승인 시: 원본 레코드 상태 자동 업데이트

반려 클릭:
1. 반려 사유 입력 모달 노출 (필수)
2. approvals.status = 'REJECTED'
3. 상신자에게 반려 알림톡 발송
4. 원본 레코드 상태 → 결재반려로 업데이트
```
