## 1. 대시보드 (DSH)

### DSH01-V 경영책임자 대시보드

| 항목 | 내용 |
|------|------|
| 화면 ID | DSH01-V |
| URL | `/` (기본 진입) |
| 접근 권한 | CEO |
| 진입 경로 | 로그인 후 자동 진입 / GNB > 대시보드 |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 역할 탭: [경영책임자] [관리자] [근로자]              │
│                                                    │
│ 이행률 요약 카드                                    │
│ 완료 N건 | 진행중 N건 | 계획 N건 | 전체 이행률 %    │
│                                                    │
│ [목표/KPI 달성률] [아차사고·작업중지 추이]           │
│                                                    │
│ [개선조치 현황]   [의견청취 현황]                   │
│                                                    │
│ [교육 현황]       [내 할일]                         │
└────────────────────────────────────────────────────┘
```

#### 위젯별 데이터 정의

**이행률 요약 카드**

| 표시 항목 | 데이터 소스 | 산출 방식 |
|----------|-----------|---------|
| 완료 건수 | `compliance_tasks.status = 'DONE'` | COUNT |
| 진행중 건수 | `compliance_tasks.status = 'IN_PROGRESS'` | COUNT |
| 계획 건수 | `compliance_tasks.status = 'SCHEDULED'` | COUNT |
| 전체 이행률 | 완료 / 전체 × 100 | % |

- 조회 범위: 로그인한 사용자의 `company_id` 기준 전체 사업장
- 기간: 현재 연도 기준

**목표/KPI 달성률**

| 표시 항목 | 데이터 소스 | 비고 |
|----------|-----------|------|
| KPI명 | `safety_kpis.name` | |
| 달성률 | `safety_kpis.current / safety_kpis.target × 100` | % |
| 목표값 / 현재값 | `safety_kpis.target` / `current` | |

- [바로가기] → POL03-S (안전경영방침 > 목표·KPI 탭)

**아차사고·작업중지 발생 추이**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 아차사고 | `opinion_requests.category = 'near_miss'` 월별 COUNT |
| 작업중지 | `opinion_requests.category = 'work_stop'` 월별 COUNT |

- 기간 필터: 6개월 / 12개월 선택
- 차트 유형: 라인 차트

**개선조치 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 대기중 | `improvements.status = 'PENDING'` COUNT |
| 조치중 | `improvements.status = 'IN_PROGRESS'` COUNT |
| 조치완료 | `improvements.status = 'DONE'` COUNT |

- 목록: 최근 3건 카드 표시 (완료예정일 / 조치내용 / 담당자)
- [바로가기] → IMP01-L

**의견청취 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 접수 | `opinion_requests.status = 'RECEIVED'` COUNT |
| 검토중 | `opinion_requests.status = 'REVIEWING'` COUNT |
| 조치중 | `opinion_requests.status = 'IN_ACTION'` COUNT |
| 반려 | `opinion_requests.status = 'REJECTED'` COUNT |
| 조치완료 | `opinion_requests.status = 'DONE'` COUNT |

- 분류별 배지 필터 (청취조사 / 안전제안 / 위험신고)
- 목록: 최근 3건 카드 표시
- [바로가기] → OPN01-L

**교육 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 교육명 | `safety_education_schedules.name` |
| 이수율 | `완료 인원 / 대상 인원 × 100` % |

- [바로가기] → EDU01-V

**내 할일 (미니 뷰)**

- TSK01-L의 축약 버전 (최근 5건)
- 탭: 문서업무 / 결재대기 / 전자서명
- [바로가기] → TSK01-L

#### 연계 화면

| 위젯 | 연계 화면 | 방향 |
|------|---------|------|
| 목표/KPI | POL03-S | 바로가기 |
| 개선조치 | IMP01-L | 바로가기 |
| 의견청취 | OPN01-L | 바로가기 |
| 교육현황 | EDU01-V | 바로가기 |
| 내 할일 | TSK01-L | 바로가기 |

┌────────────────────────────────────────────────────┐
│ 역할 탭: [경영책임자] [관리자] [근로자]              │
│                                                    │
│ 내 교육 현황 (분기별 달성률)                        │
│                                                    │
│ [내 할일]                                          │
│                                                    │
│ 바로가기 버튼                                       │

└────────────────────────────────────────────────────┘
```

#### 위젯별 데이터 정의

**내 교육 현황**

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 달성률 | `(TBM시간 + 교육시간) / 법정이수시간 × 100` % |
| 법정 이수시간 | `legal_education_hours.required` |
| 정기교육 이수시간 | `education_records.hours` WHERE `user_id = 본인` |
| TBM 인정시간 | `tbm_records.duration` WHERE `user_id = 본인` |

- 기간 탭: 26년 1분기 / 26년 2분기 / 25년 4분기

**바로가기 버튼 동작**

| 버튼 | 동작 |
|------|------|
| 아차사고 신고 | OPN01-F (category='near_miss' 자동 선택) |
| 작업중지권 행사 | OPN01-F (category='work_stop' 자동 선택) |
| 안전제안 | OPN01-F (category='suggestion' 자동 선택) |

#### 대시보드 탭 전환 로직

```
로그인한 사용자의 role 확인
- role = 'CEO' → DSH01-V 기본 노출
- role in ('GM', 'SM', 'SHM') → DSH02-V 기본 노출
- role in ('WKR', 'SUB') → DSH03-V 기본 노출

탭 전환 시 → 해당 뷰로 전환
※ 권한 없는 탭은 비활성화 (노출은 되나 클릭 불가)
```

## 2. 내 업무 (TSK)

### TSK01-L 전체 탭

| 항목 | 내용 |
|------|------|
| 화면 ID | TSK01-L |
| URL | `/my-tasks` |
| 접근 권한 | 전체 |
| 기본 진입 탭 | 전체 |

#### 레이아웃 구성

```
┌────────────────────────────────────────────────────┐
│ 내 업무                                            │
│ 요약 카드: 해야할일 N | 기한초과 N | 완료 N | 진행률 % │
│                                                    │
│ [전체] [문서업무] [결재대기] [전자서명] [완료]        │ ← 탭
│                                                    │
│ 필터: [유형▼] [상태▼]                               │
│                                                    │
│ 기한초과 업무 N건 묶음 (조건부)                     │
│ ─────────────────────────────────────────────     │
│ 업무카드 목록                                       │
│ [유형배지] [상태배지] 업무명                         │
│ 사업장 · 담당자 · 기한                               │
│ [처리하기 / 상세보기 / 결재하기 / 서명하기]           │
│                                                    │
│ [페이지네이션]                                      │
└────────────────────────────────────────────────────┘
```

#### 요약 카드 데이터 정의

| 카드 | 데이터 소스 | 조건 |
|------|-----------|------|
| 해야할일 | `tasks` | `assignee_id = 본인` AND `status != 'DONE'` COUNT |
| 기한초과 | `tasks` | `assignee_id = 본인` AND `due_date < NOW()` AND `status != 'DONE'` COUNT |
| 완료 | `tasks` | `assignee_id = 본인` AND `status = 'DONE'` COUNT (최근 30일) |
| 진행률 | 완료 / 전체 × 100 | |

#### tasks 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `task_type` | ENUM | `evaluation` `tbm` `inspection` `education` `improvement` `compliance` `approval` `signature` `subcontractor_eval` |
| `title` | VARCHAR(200) | 업무명 |
| `assignee_id` | UUID FK | 담당자 |
| `reference_id` | UUID | 연결된 원본 레코드 ID |
| `reference_type` | VARCHAR(50) | 원본 테이블명 |
| `site_id` | UUID FK | 사업장 |
| `due_date` | DATE | 기한 |
| `status` | ENUM | `PENDING` `IN_PROGRESS` `DONE` `OVERDUE` |
| `created_at` | TIMESTAMP | |

#### 업무카드 구성 요소

| 요소 | 내용 |
|------|------|
| 유형 배지 | task_type 기반 (위험성평가 / TBM / 안전점검 / 개선조치 등) |
| 상태 배지 | 대기 / 진행중 / 기한임박 / 기한초과 / 완료 |
| D-N 표시 | 기한까지 남은 일수 (D-3, D-1, 오늘, 기한초과 N일) |
| 버튼 | task_type별 다름 (아래 참조) |

#### 업무 유형별 버튼 및 연계 화면

| 업무 유형 | 버튼 | 연계 화면 |
|---------|------|---------|
| 위험성평가 | 검토하기 / 작성하기 | RSK02-D |
| TBM | 상세보기 | TBM01-D |
| 안전점검 | 처리하기 | INS02-F |
| 안전보건교육 | 상세보기 | EDU02-D |
| 개선조치 | 처리하기 | IMP01-D |
| 이행관리 | 작성하기 | CMP01-D |
| 결재 (approval) | 결재하기 / 반려 / 승인 | 해당 원본 상세 |
| 전자서명 | 서명하기 | 해당 원본 상세 |
| 도급안전평가 | 평가 시작 / 평가 계속 | CON08-D |

#### 필터 정의

| 필터 | 옵션 |
|------|------|
| 업무 유형 | 전체 / 위험성평가 / TBM / 안전점검 / 안전보건교육 / 개선조치 / 이행관리 |
| 상태 | 전체 / 진행중 / 기한임박 / 기한초과 / 완료 |

#### 기한초과 업무 묶음 표시 조건

```
due_date < NOW() AND status != 'DONE' AND count > 0
→ 목록 상단에 별도 섹션으로 그룹화하여 표시
→ 배경색: 빨간색 계열 경고 표시
```

### TSK03-L 결재대기 탭

| 항목 | 내용 |
|------|------|
| 화면 ID | TSK03-L |
| URL | `/my-tasks?tab=approval` |
| 접근 권한 | CEO, GM, SM |

#### 업무카드 구성

| 요소 | 내용 |
|------|------|
| 업무명 | `approvals.title` |
| 상신일 | `approvals.submitted_at` |
| 상신자 | `users.name` (submitter_id join) |
| 버튼 | [반려] [승인] |

#### approvals 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `approval_type` | VARCHAR(50) | 결재 유형 |
| `title` | VARCHAR(200) | |
| `reference_id` | UUID | 원본 레코드 ID |
| `submitter_id` | UUID FK | 상신자 |
| `current_approver_id` | UUID FK | 현재 결재자 |
| `status` | ENUM | `PENDING` `APPROVED` `REJECTED` |
| `submitted_at` | TIMESTAMP | |

#### 결재 처리 동작

```
승인 클릭:
1. approvals.status = 'APPROVED'
2. 결재라인 다음 결재자에게 업무 배정
3. 마지막 결재자 승인 시: 원본 레코드 상태 자동 업데이트

반려 클릭:
1. 반려 사유 입력 모달 노출 (필수)
2. approvals.status = 'REJECTED'
3. 상신자에게 반려 알림톡 발송
4. 원본 레코드 상태 → 결재반려로 업데이트
```

서명하기 클릭:
1. 서명 화면으로 이동 (원본 문서 컨텍스트)
2. 서명 입력 (터치/마우스 드로잉 또는 이름 입력)
3. 서명 완료:
   a. signature_records 테이블에 INSERT
   b. 서명 이미지 저장
   c. signature_requests.status = 'SIGNED'
   d. 요청자에게 완료 알림
```
