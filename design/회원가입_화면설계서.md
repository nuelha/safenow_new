## 1. 설계 원칙

### 핵심 구조 변경

| 항목              | v1.0 (폐기)             | v2.0 (현재)                                                       |
| ----------------- | ----------------------- | ----------------------------------------------------------------- |
| 가입 시 유형 선택 | 가입 초반에 유형 분기   | **없음** — 모두 동일한 가입 플로우                                |
| 사업자 등록 시점  | 가입 중 함께 진행       | **가입 후** 사업장 선택 화면에서 별도 추가                        |
| 초대 수락 방식    | 초대 링크로만 가입 가능 | **미리 가입 후 수락** 또는 **초대 링크로 가입 후 수락** 모두 지원 |

### 통합 원칙

- **회원가입은 모든 유형이 동일**: 본인인증 → 계정정보 → 사업장 선택 화면
- **사업자 추가는 사업장 선택 화면에서**: `+ 사업자 추가` 버튼으로 구독사업자 등록
- **초대는 사업장 선택 화면에서 수락**: 로그인 상태면 초대 카드 노출 → 그 자리에서 수락
- **전화번호 + 본인인증은 모든 유형 필수**

## 2. 화면 ID 정의

| 화면 ID   | 화면명                    | 진입 조건                              |
| --------- | ------------------------- | -------------------------------------- |
| `REG-01`  | STEP 1. 본인인증          | 회원가입 시작 (전체 동일)              |
| `REG-02`  | STEP 2. 계정정보          | REG-01 완료 후                         |
| `REG-03`  | 가입 완료                 | REG-02 완료 후                         |
| `BIZ-01`  | 사업자 추가 — 사업자 정보 | 사업장 선택 화면 > + 사업자 추가       |
| `BIZ-02`  | 사업자 추가 — 사업장 생성 | BIZ-01 완료 후                         |
| `BIZ-03`  | 사업자 추가 — 구독 결제   | BIZ-02 완료 후                         |
| `INV-01`  | 초대 수락 확인            | 사업장 선택 화면 초대 카드 > 수락 클릭 |
| `LOGIN-S` | 사업장 선택 화면          | 로그인 완료 후 항상 / 가입 완료 후     |
| `ERR-01`  | 초대 링크 만료            | 만료된 토큰 접근 시                    |

## 3. 전체 플로우 요약

### 3-1. 신규 회원가입 (모든 유형 동일)

```
/register
  → REG-01 본인인증 (전화번호 + SMS 인증)
  → REG-02 계정정보 (이메일 / 비밀번호 / 약관 동의)
  → REG-03 가입 완료
  → LOGIN-S 사업장 선택 화면 (초기: 빈 상태)
```

### 3-2. 구독사업자 등록 (가입 후)

```
LOGIN-S (사업장 선택 화면)
  → [+ 사업자 추가] 클릭
  → BIZ-01 사업자 정보 (사업자번호 / 사업자명 / 대표자명)
  → BIZ-02 사업장 생성 (사업장명 / 주소 / 업종 / 상시근로자수)
  → BIZ-03 구독 결제 (요금제 선택 / 결제)
  → LOGIN-S 로 복귀 (등록된 사업자 목록에 추가됨)
```

### 3-3. 초대 수락 — 이미 가입된 경우

```
초대 링크 클릭
  → 로그인 여부 확인
  ├── 로그인 상태: LOGIN-S로 이동 + 초대 카드 자동 노출
  └── 비로그인 상태: 로그인 화면 → 로그인 → LOGIN-S + 초대 카드 노출

LOGIN-S 초대 카드
  → [수락] 클릭 → INV-01 수락 확인 → 목록에 추가됨
  → [거절] 클릭 → 초대 카드 제거 (재발송 요청 가능)
```

### 3-4. 초대 수락 — 미가입인 경우

```
초대 링크 클릭
  → 비로그인 상태 감지 → 가입 유도 화면
  → REG-01 본인인증
  → REG-02 계정정보
  → REG-03 가입 완료
  → LOGIN-S (초대 카드 자동 노출됨)
  → [수락] 클릭 → 목록에 추가됨
```

## 4. REG-01 STEP 1. 본인인증

| 항목      | 내용                                                |
| --------- | --------------------------------------------------- |
| 화면 ID   | REG-01                                              |
| URL       | `/register`                                         |
| STEP 표시 | [1 본인인증] → [2 계정정보]                         |
| 진입 경로 | 로그인 화면 > 회원가입 / 초대 링크 (미가입 감지 시) |

### 화면 구성

```
안전NOW 회원가입

[1 본인인증] ──── [2 계정정보]

이름
[                              ]

휴대폰 번호
[010                          ] [인증번호 받기]

인증번호
[      ]  남은 시간: 2:47    [재발송]

[인증 확인]

───────────────────────────────
이미 계정이 있으신가요? [로그인하기]
```

### 입력 항목 정의

| 필드        | 타입 | 필수 | 유효성             | 비고                  |
| ----------- | ---- | ---- | ------------------ | --------------------- |
| 이름        | TEXT | ✅   | 한글/영문 2자 이상 |                       |
| 휴대폰 번호 | TEL  | ✅   | 010-XXXX-XXXX      | 중복 체크 (아래 참조) |
| 인증번호    | TEXT | ✅   | 숫자 6자리         | 유효시간 3분          |

### 휴대폰 번호 중복 체크

```
[인증번호 받기] 클릭 전 users.phone 중복 확인:

이미 가입된 번호:
→ "이미 가입된 번호입니다."
→ [로그인하기] 버튼 노출, 인증 진행 불가

초대 링크 진입 + 이미 가입된 번호:
→ "이미 가입된 계정이 있습니다. 로그인 후 초대를 수락해 주세요."
→ [로그인하기] 버튼 → 로그인 → LOGIN-S (초대 카드 자동 노출)
```

### SMS 인증 동작

```
[인증번호 받기] 클릭:
1. 6자리 코드 생성 → sms_verifications INSERT
2. SMS 발송: "[안전NOW] 인증번호 {code} (3분 이내 입력)"
3. 타이머 시작 (3:00 카운트다운)
4. [재발송] 버튼: 60초 후 활성화

[인증 확인] 클릭:
1. 코드 일치 + 유효시간 내: 인증 완료 → 필드 잠금 + "✅ 인증 완료"
2. 코드 불일치: "인증번호가 올바르지 않습니다."
3. 유효시간 초과: "인증 시간이 만료되었습니다. 재발송 해주세요."
```

### sms_verifications 테이블

| 컬럼           | 타입              | 설명                   |
| -------------- | ----------------- | ---------------------- |
| `id`           | UUID PK           |                        |
| `phone`        | VARCHAR(20)       |                        |
| `code`         | VARCHAR(6)        |                        |
| `verified`     | BOOLEAN           |                        |
| `invite_token` | VARCHAR(100) NULL | 초대 링크 진입 시 저장 |
| `expires_at`   | TIMESTAMP         | NOW() + 3분            |
| `created_at`   | TIMESTAMP         |                        |

### 하단 버튼

| 버튼 | 동작        | 조건                  |
| ---- | ----------- | --------------------- |
| 다음 | REG-02 이동 | 인증 완료 시만 활성화 |


### [가입 완료] 저장 동작

```

- 이메일 유효 + 중복 없음

- 이용약관 + 개인정보처리방침 동의 체크

저장 실행:
1. users INSERT
   (name, phone, email, password_hash, marketing_agreed)
2. 초대 링크 진입인 경우: sms_verifications.invite_token 읽어서 유지
3. REG-03으로 이동
```

### 초대 카드 노출 조건

```sql
-- 해당 사용자에게 pending 상태 초대 조회
SELECT inv.*
FROM invite_tokens inv
WHERE inv.phone = :user_phone
  AND inv.status = 'pending'
  AND inv.expires_at > NOW()
ORDER BY inv.created_at DESC
```

> 로그인 후 항상 확인. 초대 카드는 사업장 목록 상단에 별도 섹션으로 표시.

### 사업장명 옆 배지 표시 규칙

`company_type = 'principal'` 인 경우: 배지 없음 (기본 사업장)
`company_type = 'subcontractor'` 인 경우: `[도급업체]` 배지 (주황)
`company_type = 'consulting'` 인 경우: `[컨설팅]` 배지 (보라)

> 배지는 구분 컬럼이 아닌 **사업장명 텍스트 옆에 인라인**으로 표시.
> 대부분의 사용자(원청 단일 소속)는 배지 없이 깔끔하게 표시됨.

### [입장하기] 라우팅 분기

```
입장하기 클릭 시 → session.context 설정 후 라우팅

company_type = 'principal'
  role = 'CEO'          → DSH01-V
  role in (GM,SM,SHM)   → DSH02-V
  role = 'WKR'          → DSH03-V

company_type = 'subcontractor' or 'consulting'
  → DSH04-V (도급/컨설팅 간소 대시보드)
```

### [+ 사업자 추가] 버튼

```
클릭 시 → BIZ-01 (사업자 추가 — 사업자 정보)로 이동
```

```
  → 로그인 완료 → LOGIN-S (초대 카드 자동 노출) → INV-01 수락

  → BIZ-01 → BIZ-02 → BIZ-03 → LOGIN-S
```
