| 코드 | 표시명 | 색상 |
|------|--------|------|
| `EVALUATING` | 평가중 | 회색 |

#### 4단계 진행단계 표시

원형 스텝 인디케이터 (✓완료 / 숫자·색상으로 현재 강조)

| 단계 | 내용 | 완료 조건 |
|------|------|---------|
| 1 | 계약 기본정보 | 저장 완료 |
| 2 | 수급업체 평가 | 전체 업체 평가완료 + 평가자 미지정 0건 |
| 3 | 선정 + 결재 | 결재 완료 |
| 4 | 초대 + 서약 | 전체 선정 업체 서약 완료 |

### CON05-F STEP 2. 수급업체 평가

| 항목 | 내용 |
|------|------|
| 화면 ID | CON05-F |
| URL | `/contractor/:id/step2` |
| 접근 권한 | SHM+ (진행현황) / 지정된 평가자 (평가 작성) |

#### 핵심 역할 분리

| 역할 | 담당 업무 | 이 화면 접근 |
|------|---------|-----------|
| 계약 담당자 | 업체 추가 + 평가자 지정 + 진행현황 모니터링 | ✅ 전체 |
| 평가자 (안전담당자) | 실제 평가 작성 및 완료 처리 | CON08-D (내 업무에서 진입) |

> ⚠ 계약 담당자는 평가 버튼 직접 접근 불가

#### 화면 상단 구성

```
평가서 선택 영역
선택된 평가서명 + 평가항목수 + 만점 + 적격기준
[평가서 변경] → CON05-M3
```

#### 수급업체 목록 컬럼

| 컬럼 | 데이터 소스 | 관리 버튼 조건 |
|------|-----------|------------|
| 업체명 | `subcontractors.company_name` | |
| 사업자번호 | `subcontractors.business_number` | |
| 대표자 | `subcontractors.ceo_name` | |
| 관리계획서 | `management_plan_submitted` | 제출/미제출 |
| 평가자 | `evaluator_id → users.name` | 미지정 시 [평가자 지정] 버튼 |
| 평가점수 | `subcontractor_evaluations.score` | 완료 전: — |
| 결과 | `evaluation_result` | 적격/부적격/대기 |
| 상태 | `evaluation_status` | |
| 관리 | — | 아래 조건 참조 |

**관리 버튼 상태별**

| 조건 | 버튼 |
|------|------|
| 평가자 미지정 | [평가자 지정] → CON05-M2 |
| 평가자 지정됨 + 미완료 | [평가 진행중] (비활성) |
| 평가 완료 | [평가수정] (계약담당자: 조회만) |

#### 진행 조건

```
다음(STEP 3) 버튼 활성화:
1. 등록된 업체 수 >= 1
2. 모든 업체 evaluation_status = 'DONE'
3. 모든 업체 evaluator_id IS NOT NULL

미충족 시:
"전체 업체 평가가 완료되어야 다음 단계로 진행할 수 있습니다.
 (현재 N/M 완료, 평가자 미지정 K건)"
```

#### 평가 유형별 평가자 단위

| 평가 유형 | 단위 | 설정 위치 |
|---------|------|---------|
| 선정평가 | **업체별** | CON05-M / CON05-M2 |
| 정기평가 | **계약 단위** 기본 + 업체별 변경 가능 | CON04-F `default_evaluator_id` |
| 종료평가 | **계약 단위** 기본 + 업체별 변경 가능 | 동일 |

### CON05-M 업체 추가 모달

| 항목 | 내용 |

#### 탭 A. 가입 업체 검색

- 검색 대상: `companies.company_type = 'subcontractor'`

- 결과: 업체명 / 사업자번호 / 대표자 / 가입일 / [선택]
- 선택 후 평가자 지정 → 저장

#### 탭 B. 신규 업체 입력

| 필드 | 타입 | 필수 | 유효성 |
|------|------|------|------|
| 업체명 | TEXT | ✅ | |
| 사업자등록번호 | TEXT | ✅ | 10자리. 중복 체크 필수 |
| 대표자명 | TEXT | ✅ | |
| 업종 | SELECT | ✅ | |
| 담당자명 | TEXT | ✅ | 알림톡 수신자 |
| 담당자 연락처 | TEL | ✅ | |
| 담당자 이메일 | EMAIL | ❌ | |
| 안전보건관리계획서 | FILE | ❌ | |
| **평가자** | USER_SEARCH | ✅ | SHM+ 권한 사용자만 |

**사업자번호 중복 체크**

```
입력한 business_number로 companies 테이블 조회:
1. company_type='principal' → "이미 구독사업자로 등록된 번호" (진행 불가)
2. company_type='subcontractor' → "이미 가입된 업체. 탭 A에서 검색하세요"
3. 해당 없음 → 정상 입력 진행
```

**저장 동작**

```
저장 클릭:
1. subcontractors INSERT (connection_status='unlinked')
2. subcontractor_evaluations INSERT (status='PENDING')
3. tasks INSERT (task_type='subcontractor_eval', assignee_id=evaluator_id)
4. 평가자에게 알림톡 발송:
   "[안전NOW] {계약명} 수급업체 안전평가를 진행해 주세요.
    업체: {업체명} / 기한: {평가기한}"
```

#### 노출 조건 (SQL)

```sql
SELECT * FROM documents
WHERE company_id = :company_id
  AND management_type = 'form'
  AND document_type IN ('evaluation', 'checklist')
  AND status = 'ACTIVE'
ORDER BY created_at DESC
```

### CON06-F STEP 3. 선정 + 결재

| 항목 | 내용 |
|------|------|

#### 적격업체 선정 테이블

| 컬럼 | 비고 |
|------|------|
| 체크박스 | 적격: 기본 체크 / 부적격: 비활성 |
| 업체명 | |
| 사업자번호 | |
| 평가점수 | |
| 결과 | 적격/부적격 배지 |
| 선정 | 선정/선정불가 배지 |

#### 선정 규칙

```
- result='fail': 체크박스 비활성 (선정 불가)
- result='pass': 기본값 체크, 개별 해제 가능
- 최소 1개 이상 선택 후 결재 상신 가능
- 선정된 전체 업체에 결재 1건으로 일괄 상신
```

#### 결재선 자동 구성

| 역할 | 자동 배정 |
|------|---------|
| 상신자 | 로그인한 SHM |
| 중간 결재 | GM |
| 최종 결재 | CEO |

#### 선정 완료 → 결재 상신 동작

```
선정 완료 버튼 클릭:
1. subcontractors.is_selected 업데이트
2. approvals INSERT
   approval_type: 'contract_selection'
   reference_id: contract_id
3. contracts.step = 3, status = 'REVIEWING'
4. CEO/GM 내 업무 결재대기에 자동 배정
5. 알림톡: 결재 요청 알림
```

#### 결재 완료 후 자동 동작

```
결재 완료 ⟳:
1. contracts.step = 4, status = 'INVITING'
2. 선정된 subcontractors 각각:
   a. connection_status = 'unlinked' 또는 'pending_invite':
      → invite_token 생성
      → invite_expires_at = NOW() + 14days
      → connection_status = 'pending_invite'
      → 미가입 업체용 알림톡 발송
   b. connection_status = 'connected' 또는 'joined':
      → 기가입 업체용 소속 추가 알림톡 발송
3. STEP 4(CON07-F)로 자동 이동
```
