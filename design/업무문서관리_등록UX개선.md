# 업무문서관리 — 문서 등록 UX 개선 개발 스펙

> 작성일: 2026-04-29
> 관련 메뉴: 업무문서관리
> 참고 프로토타입: https://nuelha.github.io/safenow_new/pages/documents.html
> 우선순위: UX 개선

---

## 1. 개요

### 문제

현재 업무문서관리 UX에서 두 가지 혼란이 발생함.

**문제 A — 표준문서 선택 위치**
파일 업로드 화면 안에 표준문서 선택이 함께 노출됨.
사용자가 파일을 올리려는 맥락과 표준문서를 선택하는 맥락이 달라서 혼란스럽고,
표준문서가 많을수록 선택 UI가 복잡해짐.

**문제 B — 최초 등록 vs 버전 업데이트 구분 없음**
"+ 문서 등록" 버튼이 최초 등록과 버전 업데이트를 구분하지 않음.
이미 등록된 문서를 업데이트할 때도 새 문서를 만드는 것처럼 느껴짐.
결과적으로 같은 문서가 여러 개 중복 등록될 위험이 있음.

---

## 2. 개선 방향

### 개선 A — 표준문서 접근 경로 분리

**변경 전**

```
문서 등록 화면
└── 파일 업로드 탭
    └── 표준문서에서 선택 (목록 노출)
```

**변경 후**

```
문서 등록 화면
└── 파일 업로드만 (표준문서 선택 제거)

표준문서 탭 (별도)
└── 표준문서 목록
    └── 가져오기 버튼 → 우리 회사 문서로 복사 후 편집
```

표준문서는 표준문서 탭에서 **가져오기** 버튼으로만 접근.
가져오기 시 우리 회사 문서 목록에 복사본이 생성되고,
이후 회사 상황에 맞게 수정 가능.

---

### 개선 B — 최초 등록 / 버전 업데이트 진입 경로 분리

**변경 전**

```
목록 화면 > + 문서 등록 → 파일 업로드 + 모든 정보 입력
(최초 등록인지 업데이트인지 구분 없음)
```

**변경 후**

```
최초 등록 경로
목록 화면 > + 새 문서 등록 → 파일 업로드 + 문서 정보 + 권한 설정

버전 업데이트 경로
목록 화면 > 기존 문서 클릭 > 상세 화면 > 새 버전 등록 버튼
→ 파일만 다시 업로드 (기존 문서 정보 유지)
→ 버전 히스토리에 추가
```

## 3. 화면별 변경사항 상세

---

### 3-1. 목록 화면

**버튼 레이블 변경**

| 변경 전       | 변경 후          |
| ------------- | ---------------- |
| `+ 문서 등록` | `+ 새 문서 등록` |

**레이블 변경 이유**: "등록"이라는 단어가 처음 만드는 행위임을 명확히 함.

---

### 3-2. 새 문서 등록 플로우 (최초 등록)

**목록 화면 > + 새 문서 등록 클릭 시 진입**

```
STEP 1. 관리 방식 선택
  [📄 파일형]  [📝 데이터입력형]

STEP 2-A (파일형)
  파일 업로드 영역
  - 드래그 앤 드롭 또는 클릭
  - 지원 형식: PDF, Word, Excel, PPT, 이미지 (최대 50MB)
  - ※ 표준문서 선택 제거

STEP 2-B (데이터입력형)
  양식 구성
  - 문서 유형 선택 (평가표/회의록/기록부/확인서/체크리스트)
  - 섹션/항목 구성

STEP 3. 문서 정보
  - 문서명 (필수)
  - 카테고리 (필수)
  - 문서 유형 (필수)
  - 법적 근거 (선택)
  - 설명 (선택)
  - 태그 (선택)
  - 버전 번호 (기본값: v1.0)
  - 버전 변경 사유 (기본값: "최초 등록")

STEP 4. 권한 설정
  - 공개 범위 (전체/관리자/특정 인원)
  - 이행관리 연결 (토글)
```

---

### 3-3. 문서 상세 화면

기존 "수정" 버튼을 두 가지로 분리.

**변경 전**

```
[다운로드]  [수정]
```

**변경 후**

```
[다운로드]  [정보 수정]  [새 버전 등록]
```

| 버튼         | 용도               | 변경 가능 항목                                                             |
| ------------ | ------------------ | -------------------------------------------------------------------------- |
| 정보 수정    | 메타데이터 변경    | 문서명, 카테고리, 설명, 태그, 법적근거, 공개범위                           |
| 새 버전 등록 | 파일/내용 업데이트 | 파일 교체 (파일형) 또는 데이터 수정 (데이터입력형) + 버전 번호 + 변경 사유 |

---

### 3-4. 새 버전 등록 플로우 (기존 문서 업데이트)

**문서 상세 > 새 버전 등록 클릭 시 진입**

```
현재 버전 정보 표시 (읽기전용)
문서명: 위험성평가 절차서
현재 버전: v2.1 (2026-04-15)

---

[파일형인 경우]
새 파일 업로드
- 드래그 앤 드롭 또는 클릭
- 현재 파일 정보 표시 (참고용)

[데이터입력형인 경우]
기존 데이터 불러와서 수정

---

버전 정보
- 버전 번호 (자동 증가 제안: v2.1 → v2.2, 수동 변경 가능)
- 변경 사유 (필수)
  예: "법령 개정 반영", "체크리스트 항목 추가"

---

[취소]  [새 버전 등록]
```

등록 완료 시:

- 목록에서 버전 번호 업데이트
- 문서 상태 → 등록완료 (개정필요 → 등록완료로 자동 전환)
- 버전 히스토리에 추가
- 이행관리 연결되어 있으면 완료 처리

---

### 3-5. 표준문서 탭 — 가져오기 플로우

**표준문서 탭 > 가져오기 클릭 시**

```
가져오기 확인 팝업

"수급업체 안전평가표 (표준)"을
우리 회사 문서로 가져옵니다.

가져온 문서는 우리 회사 문서 목록에 추가되며
회사 상황에 맞게 수정할 수 있습니다.

[취소]  [가져오기]
```

가져오기 완료 시:

- 우리 회사 문서 탭에 복사본 생성
- 문서명: `{표준문서명} (가져옴)`
- 버전: v1.0
- 상태: 등록완료
- 해당 문서 상세로 자동 이동

---

## 4. 상태 전이 명확화

문서 상태와 각 상태로 전환되는 시점을 명확히 정의.

| 상태       | 의미                                    | 전환 조건                             |
| ---------- | --------------------------------------- | ------------------------------------- |
| `등록완료` | 최신 버전이 등록된 정상 상태            | 최초 등록 시 / 새 버전 등록 시        |
| `미등록`   | 문서가 필요하지만 아직 등록되지 않음    | 법정 필수 문서 목록에 있으나 미등록   |
| `개정필요` | 법령 변경 등으로 업데이트가 필요한 상태 | 관리자 수동 설정 / 이행관리 기한 초과 |

---

## 5. 문서 목록 개정필요 안내 배너

현재 "법정 필수문서 3건 미등록" 배너는 유지.
추가로 "개정필요" 상태 문서도 안내.

```
⚠ 법정 필수문서 3건 미등록   [미등록 문서 확인]
📋 개정이 필요한 문서 1건     [확인하기]
```

## 6. 구현 시 주의사항

1. **중복 등록 방지**: 같은 카테고리에 동일한 문서명으로 새 문서 등록 시 "이미 등록된 문서가 있습니다. 새 버전을 등록하시겠습니까?" 안내 팝업 제공.

2. **버전 번호 자동 제안**: 새 버전 등록 시 현재 버전에서 0.1 증가한 값을 기본으로 제안. 메이저 업데이트(법령 전면 개정 등)는 1.0 증가 옵션 제공.

3. **개정필요 자동 감지**: 이행관리와 연결된 문서의 주기가 도래하거나, 관리자가 직접 설정한 경우에만 개정필요 상태 전환. 자동 전환은 최소화.

4. **표준문서 가져오기 후 원본 추적**: 가져온 문서는 `source_standard_doc_id` FK를 유지해 원본 표준문서와 연결. 표준문서가 업데이트되면 가져간 문서에 "원본이 업데이트되었습니다" 알림 제공 (선택).

5. **버전 히스토리 보존**: 버전 업데이트 시 이전 버전 파일/데이터는 삭제하지 않고 히스토리에 보존. 과거 버전 다운로드 항상 가능.
