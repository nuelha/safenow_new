## 1. WRK 메뉴 존재 이유 (LOGIN-S와 역할 구분)

| 화면                  | 역할                                      | 언제         |
| --------------------- | ----------------------------------------- | ------------ |
| LOGIN-S (사업장 선택) | 어느 사업장 컨텍스트로 진입할지 선택      | 세션 시작 전 |
| WRK (사업장관리)      | 사업장 설정·구성원·예산·모듈 등 운영 관리 | 세션 중      |

LOGIN-S는 진입 경로 선택이고, WRK는 설정·관리 도구예요. 두 화면은 역할이 달라 WRK는 여전히 필요해요.

## 2. 사업장 구조 정의

```
company (회사)
├── site A: 본사 (is_headquarters = true)
├── site B: 1공장 (사업소)
└── site C: 2공장 (사업소)
```

한 회사에 여러 사업장(site)이 존재할 수 있고,
사용자는 `memberships.site_id`로 특정 사업장에 소속됨.

### memberships 소속 범위 유형

| 유형             | site_id                | 의미                                   |
| ---------------- | ---------------------- | -------------------------------------- |
| 본사 전체 관할   | NULL 또는 본사 site_id | 전체 사업장 접근 가능 (GM, CEO)        |
| 사업소 단위 소속 | 특정 site_id           | 해당 사업소만 접근 가능 (SM, SHM, WKR) |

## 3. 역할별 WRK 접근 범위

### 3-1. CEO / GM (전체 관할)

| 화면                | 접근 범위                                             |
| ------------------- | ----------------------------------------------------- |
| WRK01-L 사업장 목록 | **전체 사업장 목록** 표시                             |
| WRK01-D 사업장 상세 | 목록에서 사업장 선택 후 진입 → **선택한 사업장** 상세 |
| 조직관리 구성원     | 전체 사업장 구성원 / 좌측 조직도에서 사업장 필터 가능 |
| 전담인력 현황       | **사업장별 선택** 후 조회                             |
| 인력평가            | **사업장별 선택** 후 조회                             |
| 컨설팅업체          | **사업장별 선택** 후 조회                             |
| 모듈설정            | 전체 기본값 설정 + 사업장별 현황 테이블               |
| 구독결제            | 회사 단위 구독 정보                                   |
| 예산                | **사업장별 선택** 후 조회                             |
| 휴일관리            | **사업장별 선택** 후 조회                             |

### 3-2. SM / SHM (사업소 단위 소속)

| 화면                | 접근 범위                                           |
| ------------------- | --------------------------------------------------- |
| WRK01-L 사업장 목록 | **노출 안 함** — 목록 진입 불가                     |
| WRK01-D 사업장 상세 | **본인 소속 사업장만** 자동 진입 (사업장 선택 없음) |
| 조직관리 구성원     | **본인 소속 사업장 구성원만**                       |
| 전담인력 현황       | **본인 소속 사업장만**                              |
| 인력평가            | **본인 소속 사업장만**                              |
| 컨설팅업체          | **본인 소속 사업장만**                              |
| 모듈설정            | 본사 기본값 읽기전용 + **본인 사업장 개별 설정만**  |
| 구독결제            | **노출 안 함** — 접근 불가                          |
| 예산                | **본인 소속 사업장만**                              |
| 휴일관리            | **본인 소속 사업장만**                              |

### 3-3. WKR / SUB

| 화면     | 접근 범위                                            |
| -------- | ---------------------------------------------------- |
| WRK 전체 | **노출 안 함** — GNB에서 사업장관리 메뉴 자체 비노출 |

## 4. WRK 진입 경로 분기

### GNB > 사업장관리 클릭 시

```
session.context.role 확인

CEO 또는 GM
→ WRK01-L (사업장 목록) 진입
  → 목록에서 사업장 클릭 → WRK01-D (해당 사업장 상세)

SM 또는 SHM
→ WRK01-L 건너뜀
→ WRK01-D 로 바로 진입
  (session.context.site_id 기준으로 해당 사업장 자동 로드)

WKR / SUB
→ GNB에 사업장관리 메뉴 자체 미노출
→ 직접 URL 접근 시 403 처리
```

## 5. WRK01-D 사업장 상세 — 역할별 UI 차이

SM/SHM은 WRK01-D가 곧 자신의 유일한 사업장 화면이므로,
CEO/GM과 다른 요소가 일부 존재해요.

| UI 요소               | CEO/GM                | SM/SHM                      |
| --------------------- | --------------------- | --------------------------- |
| 상단 탭 전체          | 전체 표시             | 구독결제 탭 숨김            |
| `← 목록으로` 버튼     | 표시 (WRK01-L로 이동) | **숨김** (돌아갈 목록 없음) |
| 사업장명 수정         | 가능                  | 가능                        |
| 사업장 삭제           | CEO만 가능            | 불가                        |
| 본사/사업소 구분 표시 | 표시                  | 표시 (읽기전용)             |

## 6. 조직관리 구성원 — 사업소 단위 필터링

### CEO/GM 뷰

```
좌측 조직도 트리
├── [전체]                ← 전체 사업장 통합
├── 다온산업 본사
│   ├── 안전보건팀
│   └── 생산팀
├── 다온산업 1공장
│   └── ...
└── 다온산업 2공장
    └── ...

우측 구성원 목록
→ 좌측 선택에 따라 필터링
```

### SM/SHM 뷰

```
좌측 조직도 트리
├── [전체]                ← 본인 사업장 전체 (다른 사업장 노출 안 함)
└── 다온산업 1공장         ← 본인 소속 사업장만
    ├── 안전팀
    └── 생산팀

우측 구성원 목록
→ 본인 사업장 구성원만 표시
```

## 7. 모듈설정 — 사업소 단위 개별 설정

### CEO/GM 뷰

```
전체 기본값 설정 (회사 단위)
→ 모든 사업장의 기본값으로 적용

사업장별 현황 테이블
사업장명 | 위험성평가 | TBM | 안전점검 | ... | 개별설정여부
본사     | ON        | ON  | ON      | ... | 기본값
1공장    | ON        | ON  | OFF     | ... | 개별설정됨
2공장    | ON        | ON  | ON      | ... | 기본값
```

### SM/SHM 뷰

```
본사 기본값 (읽기전용)
→ 회사 전체 기본값 표시. 수정 불가.

내 사업장 (1공장) 개별 설정
→ ON/OFF 토글 가능
→ "기본값과 다름" 표시
→ [기본값으로 되돌리기] 버튼
```

## 8. 데이터 조회 범위 — API 필터링 규칙

모든 WRK 관련 API는 다음 규칙으로 `site_id` 필터를 적용.

```
function getSiteFilter(context) {
  const { role, site_id, company_id } = context;

  if (role === 'CEO' || role === 'GM') {
    // 전체 사업장 조회 가능
    // 선택한 사업장이 있으면 해당 사업장만 필터
    return selectedSiteId
      ? { company_id, site_id: selectedSiteId }
      : { company_id };
  }

  if (role === 'SM' || role === 'SHM') {
    // 본인 소속 사업장만 강제 필터
    return { company_id, site_id };
  }

  // WKR, SUB
  return null; // 403
}
```

> ⚠ 서버에서 반드시 재검증. 클라이언트 컨텍스트만 믿으면 안 됨.

## 9. LOGIN-S 사업장 선택 화면 — 역할별 표시 방식

### CEO / GM

```
No │ 사업장명           │ 권한      │ 알림수신 │ 상태
───┼────────────────────┼──────────┼────────┼──────
 1 │ 다온산업 본사       │ 총괄관리자 │ 토글    │ [입장]
 2 │ 다온산업 1공장      │ 총괄관리자 │ 토글    │ [입장]
 3 │ 다온산업 2공장      │ 총괄관리자 │ 토글    │ [입장]
```

CEO/GM은 어느 사업장으로 입장하느냐에 따라 조회 범위가 달라짐.
본사로 입장 → 전체 집계 뷰
1공장으로 입장 → 1공장 단위 뷰

### SM / SHM (사업소 단일 소속)

```
No │ 사업장명           │ 권한       │ 알림수신 │ 상태
───┼────────────────────┼───────────┼────────┼──────
 1 │ 다온산업 1공장      │ 사업장관리자│ 토글    │ [입장]
```

소속이 1개이므로 선택지 없음.
자동 입장 처리도 가능하나 일관성을 위해 선택 화면 유지.

### WKR (사업소 단일 소속)

```
No │ 사업장명           │ 권한   │ 알림수신 │ 상태
───┼────────────────────┼───────┼────────┼──────
 1 │ 다온산업 1공장      │ 근로자  │ 토글    │ [입장]
```

## 11. 구현 시 주의사항

1. **WRK01-L 목록 접근 차단**: SM/SHM이 `/workplace` URL 직접 접근 시 → 본인 사업장 상세(`/workplace/:site_id`)로 자동 리다이렉트.

2. **WRK01-D 사업장 수정 권한**: SM/SHM은 본인 사업장 기본정보 수정 가능. 단, 사업장 삭제·본사 지정 변경·구독 관련 설정은 CEO만 가능.

3. **모듈설정 개별 변경 이력**: SM/SHM이 기본값과 다르게 설정한 경우 `site_module_overrides` 테이블에 기록. CEO/GM 뷰에서 "개별설정됨" 표시.

4. **조직도 트리 SM/SHM**: 다른 사업장 노드 자체를 숨김 (읽기전용으로도 노출 안 함).

5. **구독결제 탭**: CEO만 접근. SM/SHM이 탭 URL 직접 접근 시 403 처리.
