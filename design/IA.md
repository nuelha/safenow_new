## IA 작성 규칙

### 화면 ID 체계

```
[메뉴코드][번호]-[타입코드]

메뉴코드 (3자리)
  DSH  대시보드
  TSK  내 업무
  RSK  위험성평가
  TBM  TBM
  INS  안전점검
  EDU  안전보건교육
  POL  안전경영방침
  BGT  안전보건예산
  OPN  의견청취
  IMP  개선조치
  CMP  이행관리
  DOC  업무문서관리
  PRC  공정관리
  CON  도급관리
  WRK  사업장관리
  SUP  고객지원
  MYI  내 정보관리

번호 (2자리)
  01, 02, 03 ... 동일 메뉴 내 화면 순번

타입코드
  V    뷰 (View) — 대시보드처럼 읽기 중심 집계 화면
  L    목록 (List)
  D    상세 (Detail)
  F    등록/편집 폼 (Form)
  M    모달 (Modal) — 확인/취소 다이얼로그 포함, 팝업과 통일
  S    설정 (Setting)

예시
  DSH01-V    대시보드 — 경영책임자 뷰
  CON01-L    도급관리 — 도급계약 목록
  CON01-F    도급관리 — 도급계약 등록 STEP 1
  CON01-M    도급관리 — 업체 추가 모달
```

### 계층 구조 (4단계)

```
LV1  GNB 메뉴       예: 도급관리
LV2  탭 / 섹션      예: 도급계약 | 수급업체 | 협의체
LV3  화면           예: 목록 / 상세 / 등록 / 설정
LV4  모달 / 팝업    예: 업체 추가 모달 / 평가자 지정 모달
```

### 권한 표기

```
전체     모든 역할
CEO      경영책임자
GM       총괄관리자
SM       사업장관리자
SHM      안전보건관리자
WKR      근로자
SUB      도급업체 담당자
CON      컨설팅업체 담당자
SM+      SM 이상 (SM, GM, CEO)
SHM+     SHM 이상 (SHM, SM, GM, CEO)
```

### 조건/표기 규칙

```
(조건부)   특정 상태/권한에서만 노출
→          이동 / 진입
⟳          자동 전환 (사용자 액션 없이)
※          주의 / 특이사항
[역할]     해당 역할에만 노출되는 요소
```
