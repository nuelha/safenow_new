## 1. 개요

### 문제

현재 업무문서관리는 파일 업로드 기반이 기본 전제로 설계되어 있음.
도급 선정평가서처럼 플랫폼 안에서 항목별 데이터를 직접 입력해서 관리해야 하는 문서 유형이 존재하는데, 이를 수용하는 구조가 없음.

현재 평가서 설정 기능이 일부 구현되어 있으나 파일 업로드와 혼재되어 있어 구조가 어색함.

### 목표

문서 등록 시 **관리 방식**을 먼저 선택하도록 플로우 변경.

- **파일형**: 기존처럼 PDF/Word 등 파일 업로드
- **데이터입력형**: 플랫폼 안에서 양식을 구성하고 항목별 직접 입력

---

## 2. 관리 방식 정의

### 파일형 (File-based)

파일을 업로드해서 관리하는 방식. 기존 구조 유지.

**적합한 문서 유형**

- 절차서, 방침서, 계획서 (법령에 따라 특정 양식이 정해진 문서)
- 외부 기관에서 발급하는 문서 (점검 결과서, 선임 신고서 등)
- 별도 편집 프로그램으로 작성된 문서

**특징**

- PDF, Word, Excel, PowerPoint, 이미지 파일 지원
- 버전 관리 (업로드할 때마다 버전 추가)
- 다운로드, 미리보기 제공

---

### 데이터입력형 (Form-based)

플랫폼 안에서 양식을 구성하고 항목별로 직접 입력하는 방식.

**적합한 문서 유형**

- 평가표 (도급 선정평가서, 인력평가표 등)
- 회의록 (안전보건위원회 회의록 등)
- 기록부 (교육기록부, 점검기록 등)
- 확인서 (안전수칙 확인서, 서약서 등)

**특징**

- 플랫폼 안에서 데이터 직접 입력
- 항목별 데이터로 저장되어 검색·집계 가능
- 보고서 형태 출력 (PDF 내보내기)
- 전자서명 연동 가능
- 도급관리, 안전점검 등 다른 메뉴에서 양식으로 불러와 사용 가능

---

## 3. 문서 등록 플로우 변경

### 변경 전

```
STEP 1. 파일 업로드
STEP 2. 문서 정보 (카테고리, 유형, 설명 등)
STEP 3. 권한 설정 (공개 범위, 이행관리 연결)
```

### 변경 후

```
STEP 1. 관리 방식 선택
         ┌────────────────┐  ┌────────────────┐
         │  📄 파일형      │  │  📝 데이터입력형 │
         └────────────────┘  └────────────────┘

STEP 2-A (파일형 선택 시)     STEP 2-B (데이터입력형 선택 시)
파일 업로드                    양식 구성
                              - 섹션 추가
                              - 항목별 입력 방식 선택
                              - 점수 설정 (평가표인 경우)

STEP 3 (공통)
문서 정보 (카테고리, 유형, 설명, 법적근거, 버전)

STEP 4 (공통)
권한 설정 (공개 범위, 이행관리 연결)
```

## 4. STEP 1 — 관리 방식 선택 UI

두 가지 선택 카드 형태로 표시. 카드 클릭으로 선택.

```
┌──────────────────────────────┐  ┌──────────────────────────────┐
│  📄 파일형                    │  │  📝 데이터입력형               │
│                              │  │                              │
│  PDF, Word 등 파일을 업로드    │  │  양식을 구성하고 항목별로       │
│  해서 관리합니다               │  │  직접 입력해서 관리합니다        │
│                              │  │                              │
│  예: 절차서, 방침서, 계획서,   │  │  예: 평가표, 회의록,           │
│      선임신고서, 점검결과서     │  │      기록부, 확인서            │
└──────────────────────────────┘  └──────────────────────────────┘
```

## 5. STEP 2-B — 데이터입력형 양식 구성

### 양식 구성 요소

**섹션 (Section)**
양식을 논리적으로 나누는 단위. 섹션명 입력 후 항목 추가.

**항목 (Field)**
각 섹션 안에 들어가는 입력 단위.

| 항목 유형    | 설명             | 사용 예                |
| ------------ | ---------------- | ---------------------- |
| 텍스트       | 단행 텍스트 입력 | 담당자명, 장소         |
| 텍스트(장문) | 여러 줄 텍스트   | 협의 내용, 특이사항    |
| 숫자         | 숫자 입력        | 배점, 금액             |
| 날짜         | 날짜 선택        | 점검일, 교육일         |
| 선택         | 항목 중 선택     | 적합/부적합, 예/아니오 |
| 점수         | 숫자 + 만점 설정 | 평가 항목 배점         |
| 서명         | 전자서명 입력    | 확인자 서명            |
| 파일첨부     | 파일 업로드      | 증빙 자료              |

### 평가표 전용 설정

항목 유형에 "점수"가 포함된 경우 평가표 설정 추가 노출.

| 설정          | 내용                                   |
| ------------- | -------------------------------------- |
| 만점          | 전체 배점 합계                         |
| 적격 기준점수 | 이 점수 이상이면 적격 판정             |
| 평가 척도     | 5단계 / 3단계 / 2단계 / 직접 점수 입력 |

**점수 자동 계산**
항목별 점수 입력 시 합계 자동 계산 → 적격/부적격 자동 판정

### 양식 구성 UI 구조

```
[+ 섹션 추가]

섹션 1. 기본 정보         [섹션 삭제]
  항목 1. 업체명     [텍스트 ▼]  [필수 ☑]  [삭제]
  항목 2. 평가일자   [날짜   ▼]  [필수 ☑]  [삭제]
  [+ 항목 추가]

섹션 2. 안전관리 수준      [섹션 삭제]
  항목 1. 안전관리자 선임 여부  [점수 ▼]  만점: 20점  [삭제]
  항목 2. 안전교육 실시 여부   [점수 ▼]  만점: 20점  [삭제]
  [+ 항목 추가]

[미리보기]  [다음 →]
```

## 7. 문서 상세 화면 분기

### 파일형 상세

기존과 동일.

- 파일 미리보기 / 다운로드
- 버전 히스토리 (버전별 다운로드)
- 이행관리 연결 정보

### 데이터입력형 상세

- 양식 항목별 입력 데이터 조회 (읽기 전용)
- 수정 버튼 → 데이터 입력 화면으로 진입
- PDF 내보내기 버튼 (보고서 형태로 출력)
- 버전 히스토리 (입력 데이터 버전별 조회)
- 이행관리 연결 정보

## 8. 다른 메뉴에서 데이터입력형 양식 활용

데이터입력형으로 등록된 양식은 다른 메뉴에서 불러와 사용 가능.

| 사용처              | 활용 방식                                 |
| ------------------- | ----------------------------------------- |
| 도급관리 > 선정평가 | 평가표 유형 양식 선택 후 업체별 평가 진행 |
| 도급관리 > 정기평가 | 동일                                      |
| 도급관리 > 종료평가 | 동일                                      |
| 안전점검 > 점검지   | 체크리스트 유형 양식 선택 후 점검 진행    |
| 인력평가            | 평가표 유형 양식 선택 후 평가 진행        |

**연동 조건**: 양식 등록 시 문서 유형이 "평가표" 또는 "체크리스트"인 경우에만 해당 메뉴의 양식 선택 목록에 노출.

## 10. DB 설계 참고

### 업무문서 (documents) — 컬럼 추가/변경

```sql
-- 기존 컬럼 유지 +
management_type     ENUM('file', 'form') DEFAULT 'file'  -- 관리 방식

-- 파일형 전용
file_url            VARCHAR(500) NULLABLE
file_size           INTEGER NULLABLE
file_format         VARCHAR(20) NULLABLE

-- 데이터입력형 전용
form_schema         JSON NULLABLE       -- 양식 구조 (섹션/항목 정의)
form_data           JSON NULLABLE       -- 입력된 데이터
evaluation_config   JSON NULLABLE       -- 평가표 설정 (만점, 기준점수, 척도)
```

### 양식 스키마 JSON 구조 예시

```json
{
  "sections": [
    {
      "id": "sec_1",
      "title": "기본 정보",
      "fields": [
        { "id": "f_1", "label": "업체명", "type": "text", "required": true },
        { "id": "f_2", "label": "평가일자", "type": "date", "required": true }
      ]
    },
    {
      "id": "sec_2",
      "title": "안전관리 수준",
      "fields": [
        {
          "id": "f_3",
          "label": "안전관리자 선임 여부",
          "type": "score",
          "max_score": 20
        },
        {
          "id": "f_4",
          "label": "안전교육 실시 여부",
          "type": "score",
          "max_score": 20
        }
      ]
    }
  ],
  "evaluation": {
    "total_score": 100,
    "pass_score": 80,
    "scale": "5step"
  }
}
```

## 11. 구현 시 주의사항

1. **기존 데이터 호환**: 현재 등록된 문서는 모두 `management_type = 'file'`로 마이그레이션. 기존 평가서 설정 데이터는 `evaluation_config`로 이관.

2. **데이터입력형 버전 관리**: 파일형과 달리 데이터입력형은 "양식 구조(form_schema)"와 "입력 데이터(form_data)" 두 가지 모두 버전 관리 필요. 양식 구조가 변경되면 기존 입력 데이터와의 호환성 검토 필요.

3. **PDF 내보내기**: 데이터입력형 문서의 PDF 출력 시 회사명, 직인, 서명이 포함된 보고서 형태로 렌더링. 서버사이드 렌더링 권장.

4. **다른 메뉴에서 양식 불러오기**: 도급관리 등에서 양식 선택 시 `management_type = 'form'` AND `document_type IN ('평가표', '체크리스트')`인 문서만 필터링해서 목록 제공.

5. **양식 수정 시 경고**: 이미 입력 데이터가 있는 양식의 구조를 수정하면 "기존 입력 데이터에 영향을 줄 수 있습니다" 경고 표시. 항목 삭제는 별도 확인 필요.
