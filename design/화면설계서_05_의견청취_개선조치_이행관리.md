| 화면 ID | OPN01-L |
| URL | `/opinion` |

| 기본 진입 화면 | ✅ |

#### 레이아웃

```
┌────────────────────────────────────────────────────┐
│ 의견청취                                           │
│                                                    │
│ 필터: [분류▼] [상태▼] [기간 시작일~종료일]           │
│       [검색]                              [+ 작성] │
│                                                    │
│ 활성 필터 칩: [분류=아차사고 ✕] [2026-04 ✕]          │
│                                                    │
│ ─── 작업중지요청 (상단 고정, 빨간배지) ───           │
│ 분류 | 제목 | 사업장 | 작성자 | 등록일 | 상태        │
│                                                    │
│ ─── 일반 목록 ───────────────────────             │
│ 분류 | 제목 | 사업장 | 작성자 | 등록일 | 상태        │
└────────────────────────────────────────────────────┘
```

#### 필터 정의

| 필터 | 옵션 | 비고 |
|------|------|------|
| 분류 | 전체 / 작업중지요청 / 청취조사 / 아차사고 / 안전제안 / 위험신고 | URL `?category=` |
| 상태 | 전체 / 접수대기 / 점검중 / 개선조치중 / 완료 / 반려 | URL `?status=` |
| 기간 | 시작일~종료일 (date range) 또는 월 단축 선택(`YYYY-MM`) | URL `?startDate=&endDate=` 또는 `?month=YYYY-MM` |
| 검색 | 제목·내용 LIKE | URL `?q=` |

- **활성 필터 칩**: 적용된 필터를 칩으로 표시. 칩의 ✕ 클릭으로 개별 해제, [전체 해제] 버튼으로 모두 해제
- 작업중지요청 상단 고정은 분류·상태 필터와 **무관하게** 항상 노출 (단, 기간 필터 적용 시 그 범위 내만 표시)

#### 외부 진입 동선 (URL 쿼리)

| 진입처 | URL 예시 | 자동 적용 필터 |
|-------|---------|--------------|
| CEO 대시보드 — 아차사고 추이 막대 클릭 | `?category=near_miss&month=2026-03` | 분류 + 월 |
| CEO 대시보드 — 의견청취 카드 [바로가기] | `/opinion` | 없음 |
| 사이드바 GNB | `/opinion` | 없음 |

- 진입 후 활성 필터 칩이 즉시 반영되어 사용자가 어떤 필터로 들어왔는지 인지 가능

| 코드 | 표시명 | 배지 색상 |
|------|--------|---------|

| `near_miss` | 아차사고 | 주황 |

| 코드 | 표시명 | 색상 |
|------|--------|------|

| `INSPECTING` | 점검중 | 보라 |
| `IMPROVING` | 개선조치중 | 주황 |

#### 작업중지요청 상단 고정 로직

```
SELECT * FROM opinion_requests
WHERE site_id = :site_id
  AND category = 'work_stop'
  AND status NOT IN ('DONE', 'REJECTED')
ORDER BY created_at DESC
→ 목록 최상단에 별도 섹션으로 표시 (빨간 배경)

일반 목록:
SELECT * FROM opinion_requests
WHERE site_id = :site_id
  AND category != 'work_stop'
ORDER BY created_at DESC
```

### OPN01-D 의견청취 상세/처리

| 항목 | 내용 |
|------|------|
| 화면 ID | OPN01-D |
| URL | `/opinion/:id` |
| 접근 권한 | 전체 열람 / SHM+ 처리 |
| 진입 경로 | OPN01-L 행 클릭 |

#### 섹션 구성

```
1. 기본정보 (읽기전용)
   분류배지 / 제목 / 사업장 / 작성자 / 등록일

2. 상세내용
   작성 내용 / 첨부 사진

3. 처리 영역 (상태별 분기 — 아래 상세 정의)

4. 처리이력 타임라인
```

#### 처리 영역 상태별 분기 로직

**[접수대기] 상태**

```
조치방법 선택 (SHM+만 노출)

A. 현장점검 실시
   → 입력: 점검종류(기본: 특별점검) / 점검예정일 / 담당자 / 확인사항(선택)
   → 저장 시:
     1. opinion_requests.status = 'INSPECTING'
     2. inspections 테이블에 특별점검 자동 생성
        name: "[의견청취] {제목}"
        source_type: 'opinion'
        source_opinion_id: opinion_request_id
     3. opinion_requests.linked_inspection_id 업데이트
     4. 담당자 내 업무 자동 배정 + 알림톡

B. 개선조치 바로 생성
   → 입력: 담당자 / 완료예정일 / 개선내용
   → 저장 시:
     1. opinion_requests.status = 'IMPROVING'
     2. improvements 테이블에 자동 생성
        source_type: 'opinion'
        source_id: opinion_request_id
     3. opinion_requests.linked_improvement_id 업데이트
     4. 담당자 내 업무 자동 배정 + 알림톡

C. 반려
   → 입력: 반려사유 (필수)
   → 저장 시:
     1. opinion_requests.status = 'REJECTED'
     2. opinion_requests.reject_reason 저장
     3. 작성자에게 반려 알림톡 발송
```

**[점검중] 상태**

```
연결된 안전점검 카드 표시
점검명 / 상태 / 예정일 / [안전점검에서 보기→]

점검 완료 후 결과 분기 (inspections에서 처리, 의견청취 자동 업데이트):
  - 개선조치 필요: opinion_requests.status = 'IMPROVING'
  - 이상 없음: opinion_requests.status = 'DONE'
```

**[개선조치중] 상태**

```
연결된 개선조치 카드 표시
개선내용 / 담당자 / 완료예정일 / 상태 / [개선조치에서 보기→]

개선조치 완료 시 자동 처리:
improvements.status = 'DONE' 변경 시
→ opinion_requests.status = 'DONE' 자동 전환 ⟳
```

**[완료/반려] 상태**

```
읽기전용
처리이력 타임라인만 표시
```

#### 처리이력 타임라인 데이터

```
opinion_request_histories 테이블
- status: 변경된 상태
- action_description: 처리 내용 요약
- processed_by: 처리자
- created_at: 처리 일시
```

#### 작업중지요청 추가 필드

```
작업명 / 현장위치 / 중지사유
접수 즉시 담당 SHM에게 긴급 알림톡 발송 (긴급 표시)
```

#### 섹션 구성

```
1. 출처 정보
   출처배지 / [원본 화면으로 이동→]

3. 담당자 / 완료예정일

4. 첨부 파일 (원본 점검 사진 포함)

5. 완료 처리 섹션 (PENDING/IN_PROGRESS 상태일 때만 노출)
   완료 내용 (TEXTAREA) / 완료 증빙 파일 (FILE)
   [조치완료 처리] 버튼
```

#### 상태 전환 로직

```
대기중 → 조치중:
  담당자가 상세 화면 진입 시 자동 전환
  IN_PROGRESS 업데이트

조치중 → 조치완료:
  [조치완료 처리] 클릭
  1. improvements.status = 'DONE'
  2. improvements.completed_at = NOW()
  3. 완료 내용 / 증빙 파일 저장
  4. source_type='opinion' 인 경우:
     → opinion_requests.status = 'DONE' 자동 전환 ⟳
  5. 요청자 / SHM에게 완료 알림톡
```
