### WRK01-D 사업장 관리

| 항목 | 내용 |
|------|------|
| 화면 ID | WRK01-D |
| URL | `/workplace/:id` |
| 접근 권한 | CEO·GM (전체) / SM (담당 사업장만) |
| 기본 진입 화면 | ✅ |

#### 레이아웃

```
┌────────────────────────────────────────────────────┐
│ 사업장관리                                         │
│                                                    │
│ [조직관리] [컨설팅업체] [모듈설정]                  │ ← 탭

│                                                    │
│ 사업장 기본정보                                     │
│ 사업장명 / 주소 / 업종 / 상시근로자수               │
│ 역할 담당자: 안전관리자 / 사업장관리자 / 경영책임자   │
│                                                    │
│ [목록으로] [수정] [조직관리]                        │
└────────────────────────────────────────────────────┘
```

### WRK02-M 직원 등록 모달

| 항목 | 내용 |
|------|------|
| 화면 ID | WRK02-M |
| 접근 권한 | SM+ |
| 진입 경로 | WRK02-L > + 직원등록 버튼 |

#### 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 이름 | TEXT | ✅ | |
| 연락처 | TEL | ✅ | 알림톡 발송 번호 |
| 이메일 | EMAIL | ❌ | |
| 부서 | TEXT | ❌ | |
| 직책 | TEXT | ❌ | |
| 역할 | SELECT | ✅ | SHM / WKR 등 |

#### 저장 동작

```
등록 저장 클릭:
1. memberships INSERT (status='pending')
2. 초대 링크 생성 (invite_token)
3. 알림톡 발송:
   "[안전NOW] {사업장명}에서 구성원으로 초대되었습니다.
    ▶ 가입: {초대 링크} / 유효기간: 2주"
```

|------|------|

| 접근 권한 | SM+ |

#### 구성

```
1. 엑셀 템플릿 다운로드 버튼

3. 업로드 시 유효성 검사

5. 오류 없으면 미리보기 후 [일괄 등록] 버튼
```

### WRK03-V 전담인력 현황 (조직관리 > 전담인력 탭)

| 항목 | 내용 |
|------|------|
| 화면 ID | WRK03-V |
| URL | `/workplace/:id/organization?tab=dedicated` |
| 접근 권한 | SM+ |

#### 구성

```
업종 + 상시근로자수 기반 역할 자동 산출

역할 카드
┌─────────────────────────────────────────┐
│ 안전관리자     [충족] 1명 선임 필요        │
│ 담당자: 김안전                 [수정]     │
├─────────────────────────────────────────┤
│ 보건관리자     [미충족] 1명 선임 필요      │
│ 담당자: 미지정                 [선임]     │
└─────────────────────────────────────────┘
```

#### dedicated_personnel 테이블 주요 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID PK | |
| `site_id` | UUID FK | |
| `role_type` | VARCHAR(50) | 안전관리자/보건관리자/관리감독자 등 |
| `required_count` | INTEGER | 법적 선임 필요 수 |
| `user_id` | UUID FK NULL | 선임된 사용자 |
| `appointed_at` | DATE | |

#### 자동 산출 로직

```
업종 코드 + 상시근로자수 →
legal_requirements 테이블에서 조회
→ 역할별 선임 필요 인원 산출
→ dedicated_personnel 레코드와 비교
→ 충족/미충족 배지 표시

관리감독자:
부서장(department_head) 계정 자동 연결
(memberships.position = '부서장' 인 사용자)
```

### WRK07-S 구독결제 탭

| 항목 | 내용 |
|------|------|
| 화면 ID | WRK07-S |
| URL | `/workplace/:id/subscription` |
| 접근 권한 | CEO |

#### 현재 구독 현황 카드

| 표시 항목 | 데이터 소스 |
|----------|-----------|
| 구독 티어 | `subscriptions.tier` |
| 인원 범위 | 티어별 상한 |
| 월 요금 | `subscriptions.monthly_fee` |
| 결제일 | `subscriptions.billing_date` |
| 실제 연결 인원 | `memberships.status='active'` COUNT |

#### 버튼 및 동작

| 버튼 | 동작 |
|------|------|
| 업그레이드 | 상위 티어 선택 → 즉시 변경 + 알림톡 D-3 발송 |
| 다운그레이드 | 하위 티어 선택 → 다음 결제일 적용 (수동 처리) |
| 해지 | 해지 확인 → 당일 해지 시 전액 환불 / 이후 환불 없음 |

#### 티어 정의 (7단계)

| 티어 | 인원 범위 |
|------|---------|
| T1 | 1~30명 |
| T2 | 31~50명 |
| T3 | 51~100명 |
| T4 | 101~200명 |
| T5 | 201~300명 |
| T6 | 301~500명 |
| T7 | 501~1,000명 |
